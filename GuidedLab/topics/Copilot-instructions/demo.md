# TODO: copilot instructions
- Custom instructions = preloading your mental context.
- They work like:

    - Default parameters in code

    - Predefined mental math shortcuts

    - A personal coding style rulebook that Copilot follows automatically

| **Aspect**         | **Custom Copilot Instructions**                                                                     |
| ------------------ | --------------------------------------------------------------------------------------------------- |
| **What it is**     | A set of rules or preferences you give to GitHub Copilot to follow automatically.                   |
| **Purpose**        | Tailor the AI’s coding output to your style, libraries, and workflow.                               |
| **How it works**   | You set defaults once — every Copilot suggestion follows them unless you override.                  |
| **Analogy**        | Like setting *default parameters* in a function.                                                    |
| **Example**        | “Always use `pandas`, follow PEP8, include docstrings.” Now, “Read a CSV” auto-follows these rules. |
| **Feedback loop**  | If Copilot ignores a rule, you adjust the custom instructions.                                      |
| **Learning curve** | One-time setup, ongoing refinement as coding style changes.                                         |
